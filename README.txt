【ツール概要】
　本ツールは、テキストファイル(＝非バイナリファイル)に対するXXXツールの実行を、VSCodeから直接行えるようにするためのツールです。
　テキストファイルからWordファイルへの変換（XXXツールがWord/Excelのみ対応のため）および、XXXツールの自動操作を行います。

　VSCodeのアクティブなタブで開いているファイル(1つ)、
　もしくは「scripts/_config.ps1」の "$CHECK_TARGET_ROOT_DIR" 配下のファイル(複数)を対象に実行できます。
　
　本ツールの各種スクリプトファイルを、VSCodeのタスクに登録して実行します。

【ツール導入】
　(1)「scripts/」を任意のフォルダにコピー
　(2)「vscode-settings/tasks.json」を「%USERPROFILE%/AppData/Roaming/Code/User/」配下へコピー
　(3) (1)でコピーした「_config.ps1」の "※各自ローカルのパスに変更" のコメント記載が記載されている変数のパスを、(1)のパスに書き換え
　(4) (2)でコピーした「tasks.json」の"execute-check.bat" のパスを、(1)のフォルダパスに書き換え
　　　　※円マークは要エスケープ

【利用手順】
　(1) VSCodeにて、Ctrl + Shift + P でコマンドパレットを起動し、「Task」と入力
　　　※一度実行すれば、コマンドパレットの上部に直近で実行されたコマンドが表示されているため「Task」の入力はしなくても、矢印キー(一番上の表示以外の場合)とEnterの入力でも可。

　(2)「Task: Run Task」を選択し、「tasks.json」に記載されているタスクが表示されるので、実行したいタスクを選択する。
　　　　〇 execute-check-active-tab：
　　　　　　VSCodeのアクティブなタブで開いているファイルを対象にXXXツールを実行する。

　　　　〇 execute-check：
　　　　　　「scripts/_config.ps1」の "$CHECK_TARGET_ROOT_DIR" 配下のファイル(複数)を対象にXXXツールを実行する。

　　　※タスクを実行すると、VSCode下部にターミナルが表示される。
　　　　また、タスクの実行が完了すると、ターミナルに下記メッセージが表示されるので、邪魔であればてきとうなキーを入力して閉じる。
　　　　　「Terminal will be reused by tasks, press any key to close it.」
　　　　　「ターミナルはタスクによって再利用されます。閉じるには任意のキーを押してください。」

　(3) タスクの実行が完了すると、本ツールにより、XXXツールの「inフォルダ/forVSCode/実行日時」配下にテキストファイルから変換したWordファイルが出力され、
　　　XXXツールにより、「outフォルダ/forVSCode/実行日時」配下にそのWordファイルに対するXXXチェック結果のWordファイルが出力される。
　　　また、XXXチェック結果のWordファイルが自動で開かれる。

【トラブルシューティング】
　・VSCode下部のターミナルに、赤字でエラーメッセージが表示されるので、その内容から解析する。
　・XXXツールの自動操作周りのエラーの場合、コントロールの取得に失敗している可能性が高い。
　　ターミナルに、[Window]、[Controll]で始まるログを参考に対処する。
